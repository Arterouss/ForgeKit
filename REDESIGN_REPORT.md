# DevForge visual redesign

## Product direction

DevForge now uses a graphite, violet, and teal desktop-workspace identity. The previous high-saturation cyber-terminal treatment was removed in favor of layered glass surfaces, restrained borders, natural language labels, and a quieter typography scale.

## Rebuilt surfaces

- **Landing page** — Replaced with a focused product story: centered hero, working-space preview, workflow features, plugin ecosystem, testimonials, FAQ, and a compact footer.
- **Application shell** — New floating-style sidebar, command search bar, responsive utility controls, and a simplified local-session status bar.
- **Dashboard** — Rebuilt as a productivity home with tool search, continuation cards, quick actions, domains, favorites, and recent activity backed by the existing registry and workspace data.
- **Tool workspace** — Reframed as a native canvas with clean tabs, compact tool controls, split-view preference, sharing, fullscreen mode, and the existing input/output engine intact.
- **Shared cards and tool browsing** — Replaced the neon card treatments with a single glass-panel card language across tools and categories.

## Design system

- Obsidian canvas with soft, low-noise violet depth.
- One radius scale: 12px for controls, 16px for panels, 24–28px for major compositions.
- Geist for product UI and JetBrains Mono for code, shortcuts, and compact metadata.
- Visible keyboard focus, reduced visual density, subtle hover elevation, and Framer Motion transitions.

## Preserved behavior

The redesign keeps the route structure, tool registry and engines, plugin SDK, command palette, workspace persistence, favorites, history, collections, tabs, sharing, and fullscreen controls. The work changes presentation rather than the underlying developer workflow.
